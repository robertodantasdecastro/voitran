# Agent Lifecycle

## Entrada
- todo agente novo entra por `W28_onboard_novo_agente`;
- deve ser registrado em `AGENT_REGISTRY.md`, `AGENT_SYNC_BOARD.md` e `PROJECT_STATE.md`.

## Operacao
- toda divergencia operacional deve ser registrada no board;
- toda mudanca estrutural exige atualizacao de memoria viva;
- agentes seguem a mesma regra de segredos e runtime externo.

## Saida ou desativacao
- registrar status;
- registrar impacto em ownership;
- registrar handoff e proximo responsavel.
